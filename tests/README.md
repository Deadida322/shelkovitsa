# API Тесты для Shelkovitsa

Эта папка содержит Postman коллекцию с тестами для всех API эндпоинтов бекенда.

## Файлы

- `Shelkovitsa_API.postman_collection.json` - Postman коллекция с тестами для всех эндпоинтов

## Как использовать

### 1. Импорт коллекции в Postman

1. Откройте Postman
2. Нажмите "Import" в левом верхнем углу
3. Выберите файл `Shelkovitsa_API.postman_collection.json`
4. Коллекция будет импортирована со всеми тестами

### 2. Настройка переменных окружения

В коллекции используются следующие переменные:

- `baseUrl` - базовый URL API (по умолчанию: `http://localhost:3000`)
- `adminToken` - JWT токен администратора (для админских эндпоинтов)
- `orderId` - ID заказа (автоматически устанавливается при создании заказа)
- `userId` - ID пользователя (автоматически устанавливается при регистрации)

Для настройки переменных:

1. Откройте коллекцию в Postman
2. Перейдите на вкладку "Variables"
3. Установите нужные значения

### 3. Запуск тестов

#### Запуск всех тестов

1. Выберите коллекцию "Shelkovitsa API"
2. Нажмите "Run" (кнопка справа)
3. Выберите нужные тесты и нажмите "Run Shelkovitsa API"

#### Запуск отдельных тестов

1. Откройте нужный запрос в коллекции
2. Нажмите "Send"
3. Проверьте результаты в разделе "Test Results"

### 4. Последовательность выполнения тестов

Рекомендуется выполнять тесты в следующем порядке:

1. **Auth** → Register (создание пользователя)
2. **Auth** → Login (получение токена)
3. **Order** → Create Order (создание заказа)
4. **Order** → Get Order List (получение списка заказов)
5. Остальные тесты в любом порядке

## Структура коллекции

Коллекция организована по модулям:

- **Auth** - аутентификация и авторизация
  - Register
  - Login
  - Get Me

- **Order** - управление заказами
  - Create Order
  - Get Order List
  - Get Admin Order List
  - Change Order Status
  - Get Failed Telegram Orders
  - Retry Telegram Message

- **Product** - продукты
  - Get Product
  - Get Product Admin

- **Product Article** - артикулы продуктов
  - Get Product Article List
  - Get Product Article
  - Get Populate List
  - Get Admin Product Article List

- **Product Category** - категории продуктов
  - Get Category List
  - Create Category

- **Product Color** - цвета продуктов
  - Get Color List
  - Create Color

- **Product Size** - размеры продуктов
  - Get Size List

- **Delivery Type** - типы доставки
  - Get Delivery Type List

- **Benefit** - преимущества
  - Get Benefit List

- **User** - пользователи
  - Get Basket
  - Update Basket

## Тесты

Каждый запрос содержит автоматические тесты, которые проверяют:

- Статус код ответа
- Структуру данных ответа
- Наличие обязательных полей

Результаты тестов отображаются в разделе "Test Results" после выполнения запроса.

## Примечания

- Для админских эндпоинтов требуется установить переменную `adminToken`
- Некоторые тесты требуют предварительного выполнения других тестов (например, создание заказа требует существования продуктов)
- Убедитесь, что сервер запущен перед выполнением тестов

## Swagger документация

После запуска сервера, Swagger документация доступна по адресу:

```
http://localhost:3000/api/docs
```

Здесь вы можете увидеть все эндпоинты с подробным описанием, схемами запросов и ответов.
